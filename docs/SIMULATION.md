# 组合模拟口径说明（P4-1）

状态：`P4-1-candidate`  
稳定基线：`v3.5.1-stable`

## 定位

`simulation/` 是独立的只读计算层，用于比较当前组合与目标权重组合。它不修改持仓、不生成真实订单、不预测未来收益，也不依赖数据库、行情或 FastAPI。

P4-1 尚未接入 API 和前端。

## 输入

模拟核心接收：

- 当前总资产与现金。
- 当前基金代码、名称、市值和权重。
- 各当前基金的目标权重。
- 目标现金权重。

权重统一使用百分数 `0~100`。目标中省略的当前基金按目标权重 0 处理，即模拟清仓；未知基金代码直接拒绝。

## 校验

- 总资产必须大于 0。
- 当前现金和持仓市值不得为负。
- 当前基金代码必须唯一。
- 基金目标权重及现金权重必须位于 `0~100`。
- 基金目标权重总和 + 现金目标权重必须等于 100，允许 `0.01` 个百分点误差。
- 禁止做空和杠杆。
- 当前持仓市值 + 现金必须与总资产守恒，金额误差不得超过 0.02 元。

校验失败时抛出 `SimulationValidationError`，不会猜测或静默修正用户意图。

## 计算口径

### 目标金额与调仓差额

```text
target_value = total_asset × target_weight / 100
trade_delta = target_value - current_value
```

- 正数：模拟买入。
- 负数：模拟卖出。
- 0：保持。

金额使用 `Decimal` 和 `ROUND_HALF_UP`，输出保留 2 位小数。

### 分币尾差

每个目标桶独立四舍五入后可能产生分币尾差。模拟器将尾差分配给目标权重最大的桶；并列时按稳定的桶标识顺序选择。该规则确定、可重复，并确保目标基金金额与目标现金之和严格等于总资产。

### 单边换手率

```text
turnover_pct = 0.5 × Σ(abs(fund_trade_delta)) / total_asset × 100
```

现金差额用于资产守恒，不重复计入基金换手率。

### HHI

```text
HHI = Σ((fund_weight / 100)²)
```

现金不纳入基金 HHI。输出当前与目标 HHI、最大单基金权重及其变化。

## 提示规则

- 单基金目标权重 > 30%：集中度提示。
- 单基金目标权重 > 50%：高集中度提示。
- 目标 HHI 高于当前 HHI：集中度上升提示。
- 单边换手率 > 50%：高换手提示。

提示仅描述结构，不构成投资建议。

## 模块

- `simulation/models.py`：不可变输入输出数据结构。
- `simulation/portfolio_simulator.py`：纯计算核心。
- `simulation/adapter.py`：把既有 `calc_portfolio()` 结果转换成标准快照；仅 adapter 可以在显式调用时接触该只读边界。

## 已知限制

- 只能调整当前快照中已存在的基金。
- 不支持新增基金、手续费、税费、滑点、申赎时延或最小交易单位。
- 不估算未来净值、收益率、回撤或胜率。
- 不进行优化求解，只计算用户给定的目标权重。
- 不生成或执行真实交易。
